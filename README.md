# MOX Test App — Лисий диспетчер + Ферма кроликов

Интерактивное мини-приложение для тестового задания MOX. Оба сценария из ТЗ доступны в одном интерфейсе — переключатель в верхней части экрана.

## Стек

- Vite
- React 19
- TypeScript
- CSS (без UI-фреймворков)

## Структура проекта

```
src/
├── app/                    # переключатель сценариев, роутинг
├── shared/                 # AppShell, worklog, общие UI-компоненты
├── scenarios/
│   ├── fox/                # «Лисий диспетчер» (готов)
│   └── rabbits/            # «Ферма кроликов» (готов)
```

## Запуск

```bash
npm install
npm run dev
```

Сборка:

```bash
npm run build
npm run preview
```

0. **Переключатель сценариев** — смена лис ↔ кролики, выбор сохраняется после F5
1. **Дашборд (лисы)** — 2 лисы, 3 наблюдения, лидер `fox_001`, слайдеры меняют баллы
2. **Наблюдения** — добавить / изменить / удалить запись, сортировка, фильтры
3. **Импорт JSON** — валидный массив импортируется, невалидный показывает ошибку
4. **Лисы** — карточки и таблица, «Показать в журнале»
5. **Локации** — выбор локации, детали, переход в журнал
6. **AI Worklog** — чекпоинты и скриншоты сохраняются после F5
7. **Сброс (лисы)** — «Сбросить к стартовым» возвращает 3 наблюдения из ТЗ
8. **Дашборд (кролики)** — оценка, уверенность, слайдеры весов меняют расчёт
9. **Сигналы** — добавить / изменить / удалить запись, сортировка, фильтры
10. **Импорт JSON (кролики)** — валидный массив импортируется, невалидный показывает ошибку
11. **Локации / типы (кролики)** — детали и переход в журнал с фильтром
12. **Сброс (кролики)** — возвращает 3 сигнала из ТЗ
13. **AI Worklog (кролики)** — чекпоинты и скриншоты сохраняются после F5
14. **Баннер фильтров (кролики)** — виден на дашборде, сигналах, локациях и типах

### AI Worklog для всех пользователей (деплой)

Worklog можно «зашить» в проект, чтобы после деплоя его видели все на любых устройствах.

**Файлы встроенного worklog (раздельно по сценариям):**

| Сценарий | Файл |
|----------|------|
| Лисы | `src/scenarios/fox/data/initialWorklog.ts` |
| Кролики | `src/scenarios/rabbits/data/initialWorklog.ts` |

**Как обновить worklog перед сдачей:**

1. Заполните чекпоинты и прикрепите скриншоты на вкладке AI Worklog нужного сценария.
2. Нажмите **«Экспорт для деплоя»** — скачается JSON (`fox-dispatcher-worklog-seed.json` или `rabbit-farm-worklog-seed.json`).
3. Откройте соответствующий `initialWorklog.ts` и вставьте содержимое JSON в объект `initialWorklog` (поле `checkpoints` со скриншотами в base64).
4. Закоммитьте и задеплойте.

При **первом открытии** сайта (пустой localStorage) пользователь увидит встроенный worklog. Кнопка **«Встроенный worklog»** на вкладке сбрасывает локальные правки и снова показывает версию из репозитория.

## Как пользоваться: Лисий диспетчер

### О чём этот сценарий

Вы — лесной смотритель. В лесу живут лисы, и вы фиксируете, где их видели, какого они цвета, были ли с добычей и насколько подозрительно они себя вели. Приложение помогает понять, **что происходит в лесу**, **где больше активности** и **какая лиса сейчас выглядит наиболее подозрительной**.

Вверху экрана можно переключиться между «Лисий диспетчер» и «Ферма кроликов». Ваш выбор запоминается — после обновления страницы откроется тот же сценарий.

### Вкладки

| Вкладка | Что здесь |
|---------|-----------|
| **Дашборд** | Главная сводка: сколько лис замечено, сколько наблюдений, самая «горячая» локация, лидер по подозрительности. Здесь же — слайдеры, которыми можно настроить, что важнее при оценке. |
| **Наблюдения** | Журнал всех записей: добавить, изменить, удалить. Есть фильтры и импорт данных из JSON. |
| **Лисы** | Рейтинг лис с объяснением, почему каждая из них выглядит подозрительной. Можно смотреть карточками или таблицей. |
| **Локации** | Карта активности по местам леса. Можно выбрать локацию и посмотреть, что там происходило. |
| **AI Worklog** | Ваш дневник работы с AI: ключевые шаги, решения, скриншоты. |

### Что такое одно «наблюдение»

Каждая запись в журнале — это один случай, когда лису заметили:

- **Где** — локация (например, «Северная поляна»)
- **Какая лиса** — её ID (например, `fox_001`)
- **Цвет** — рыжая, чёрная и т.д.
- **Была ли с добычей** — да или нет
- **Подозрительность** — число от 1 до 10
- **Время** — когда видели (формат 08:20)

При первом запуске уже есть **3 наблюдения** и **2 лисы** — это стартовые данные из задания.

### Как приложение оценивает лис

Для каждого наблюдения считается **балл подозрительности**. На него влияют:

1. **Уровень подозрительности** из записи (чем выше число — тем больше балл)
2. **Наличие добычи** — если лису видели с добычей, добавляется бонус

Потом все наблюдения одной лисы складываются в **средний балл**. Лиса с самым высоким средним баллом — **лидер**, самая подозрительная на данный момент.

На стартовых данных лидер — **fox_001**: её видели дважды, один раз с добычей, с высокой подозрительностью.

### Слайдеры на дашборде

Два ползунка позволяют настроить оценку «на лету»:

- **Вес уровня подозрительности** — насколько сильно учитывать число 1–10
- **Бонус за добычу** — насколько сильно «наказывать» лису за то, что её видели с добычей

Подвинули слайдер — рейтинг лис и вся сводка **сразу пересчитываются**. Кнопка «Сбросить параметры» возвращает значения по умолчанию.

### Фильтры

Можно сузить отчёт: показать только одну лису, одну локацию, один цвет, только записи с добычей или с определённой подозрительностью. Фильтры действуют на все вкладки. Если что-то отфильтровано — появится баннер с кнопкой «Сбросить фильтры».

### Типичный сценарий

1. Откройте **Дашборд** — посмотрите сводку и лидера.
2. Подвигайте слайдеры — посмотрите, как меняется рейтинг.
3. Зайдите в **Наблюдения** — добавьте новую запись или отредактируйте существующую.
4. На вкладке **Лисы** — сравните карточки и прочитайте объяснения.
5. На вкладке **Локации** — выберите место и нажмите «Открыть в журнале», чтобы увидеть только записи оттуда.

Данные сохраняются в браузере. Кнопка **«Сбросить к стартовым»** на вкладке «Наблюдения» вернёт исходные 3 записи из задания.

## Как пользоваться: Ферма невидимых кроликов

### О чём этот сценарий

На ферме живут **невидимые кролики** — их нельзя увидеть напрямую. Но фермер замечает **косвенные признаки**: пропала морковка, появилась ямка, сработал датчик движения, кто-то шуршал в сарае. Приложение по этим сигналам оценивает, **сколько кроликов, примерно, на ферме**, **насколько можно доверять оценке** и **что делать дальше**.

### Вкладки

| Вкладка | Что здесь |
|---------|-----------|
| **Дашборд** | Главный ответ: примерное число кроликов, уверенность системы, самые влиятельные сигналы, рекомендации фермеру. Здесь же — слайдеры весов. |
| **Сигналы** | Журнал всех косвенных наблюдений: добавить, изменить, удалить. Есть фильтры и импорт JSON. |
| **Локации** | Где на ферме больше всего «тревожных» событий. |
| **Типы сигналов** | Какой вид признаков сильнее всего повлиял на оценку (морковка, ямки, датчик и т.д.). |
| **AI Worklog** | Ваш дневник работы с AI — отдельный от сценария с лисами. |

### Что такое один «сигнал»

Каждая запись — одно косвенное наблюдение:

- **Тип** — пропажа морковки, новая ямка, датчик движения или шуршание
- **Локация** — где заметили (Огород, Сарай, У забора…)
- **Количество** — сколько раз или сколько единиц (например, 5 морковок)
- **Интенсивность** — насколько сильный сигнал, от 1 до 10
- **Время** — когда произошло (формат 08:30)

При первом запуске уже есть **3 сигнала** из задания: морковка на огороде, ямки у забора, датчик в сарае.

### Как приложение считает кроликов

Каждый сигнал вносит **вклад** в общую оценку. Чем больше количество, выше интенсивность и «веснее» тип события — тем больше вклад.

Типы по умолчанию учитываются по-разному: датчик движения весит больше, чем пропажа морковки, ямка — где-то посередине. Эти пропорции можно менять слайдерами.

Все вклады складываются, результат **округляется** — получается примерное число кроликов. На стартовых данных это **около 6 кроликов**.

### Уверенность системы

Отдельно показывается **уверенность в процентах** (0–100%). Это не «точность прибора», а ответ на вопрос: **достаточно ли данных для вывода**. Уверенность выше, если:

- сигналов больше
- они разных типов
- замечены в разных местах фермы
- в среднем интенсивность повыше

На стартовых данных уверенность — **около 81%**.

### Рекомендации

На дашборде приложение подсказывает, что делать фермеру — простыми правилами, без магии:

- много пропаж морковки → защитить огород
- много ямок → укрепить забор
- сработал датчик в сарае → проверить сарай
- мало данных → собрать больше сигналов
- высокая оценка поголовья → усилить меры

Рекомендации обновляются при изменении данных.

### Слайдеры на дашборде

Пять ползунков настраивают, чему доверять больше:

- вес **пропажи морковки**
- вес **новых ямок**
- вес **датчика движения**
- вес **шуршания**
- общий **множитель интенсивности**

Подвинули слайдер — число кроликов, топ сигналов и рекомендации **пересчитываются сразу**.

### Фильтры и переходы между вкладками

Можно отфильтровать сигналы по типу, локации, интенсивности и количеству. Если на **Локациях** или **Типах сигналов** нажать на элемент — откроется журнал уже с нужным фильтром. Активные фильтры видны в баннере вверху экрана.

### Типичный сценарий

1. Откройте **Дашборд** — посмотрите оценку (~6 кроликов), уверенность и рекомендации.
2. Подвигайте слайдер «Датчик движения» — посмотрите, как меняется число.
3. Зайдите в **Сигналы** — добавьте новое событие (например, шуршание в сарае).
4. Вернитесь на **Дашборд** — оценка уже другая.
5. На вкладке **Типы сигналов** нажмите на тип — откроется отфильтрованный журнал.

Данные сохраняются в браузере. Кнопка **«Сбросить к стартовым»** вернёт исходные 3 сигнала из задания.
